# Quick Start Guide

Get the KPI Intelligence Platform running in 5 minutes!

## 🚀 Installation Methods

### Method 1: Automated Setup (Recommended)

```bash
# Clone the repository
git clone <repository-url>
cd kpi-intelligence-platform

# Run setup script (Linux/Mac)
./setup.sh

# Start the application
streamlit run app.py
```

### Method 2: Manual Setup

```bash
# Create virtual environment
python3 -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Generate sample data
python generate_sample_data.py

# Run the application
streamlit run app.py
```

### Method 3: Docker (Zero Configuration)

```bash
# Start with Docker Compose
docker-compose up -d

# View logs
docker-compose logs -f

# Stop
docker-compose down
```

## 💡 Tips for Best Results

### Data Preparation
1. **Clean column names**: Use underscores instead of spaces
2. **Consistent data types**: Ensure numeric columns don't have text
3. **Date formatting**: Use standard formats (YYYY-MM-DD, MM/DD/YYYY)
4. **Remove headers**: First row should contain column names only

### File Optimization
- **CSV preferred**: Faster loading than Excel
- **Filter before upload**: Remove unnecessary columns
- **Reasonable size**: 50-100 MB for best performance
- **UTF-8 encoding**: Ensure proper character encoding

### KPI Detection
- **Descriptive names**: Column names like `total_revenue` work better than `col1`
- **Numeric data**: Ensure amounts don't include currency symbols in cells
- **Date columns**: At least one date column enables trend analysis
- **Avoid too many columns**: 50-100 columns optimal

## 🔧 Common Issues & Solutions

### Issue: "No data rows found in file"
**Solution**: Ensure your file has a header row and at least one data row

### Issue: "File too large (max 50 MB)"
**Solution**: 
- Filter data to recent time period
- Remove unnecessary columns
- Save as CSV instead of Excel

### Issue: "No KPIs detected"
**Solution**:
- Check that you have numeric columns
- Ensure column names are descriptive
- Verify data types are correct

### Issue: Charts not displaying
**Solution**:
- Clear browser cache
- Refresh the page
- Check that you have selected valid columns
